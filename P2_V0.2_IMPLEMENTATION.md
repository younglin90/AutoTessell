## ⚠ 알려진 제약사항

### pyvista OpenFOAM 지원
- pyvista는 원시 polyMesh 형식을 완전히 지원하지 않음
- 현재: VTK 변환 실패 → neatmesh 스킵 (fallback)
- 영향: negligible (native metrics는 여전히 정확함)

### 향후 개선
1. **polyMesh → meshio 변환기 구현**
   - 사용자 정의 변환기로 VTK/Gmsh 형식 생성
   - 임의 다면체 → 사면체 분해

2. **foamToVTK 대체**
   - 기존: foamToVTK (OpenFOAM 필수)
   - 신규: pure Python polyMesh reader + pyvista/meshio writer

3. **neatmesh 결과 병합**
   - 현재: 실패 시 스킵 (graceful)
   - 향후: 보충 메트릭 CheckMeshResult에 병합
