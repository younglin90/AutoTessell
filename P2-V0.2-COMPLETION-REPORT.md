### 4. MeshQualityChecker Fallback Chain
**파일**: `core/evaluator/quality_checker.py`

#### 동작 흐름
```python
MeshQualityChecker.run(case_dir)
  ↓
try:
  _run_openfoam(case_dir)  # OpenFOAM 설치 시
catch FileNotFoundError:
  ↓
  NativeMeshChecker().run(case_dir)  # OpenFOAM 없을 시
    ↓
    parse polyMesh files (points, faces, owner, neighbour)
    ↓
    compute numpy-based metrics
    ↓
    if neatmesh_available():
      run_neatmesh() for supplementary metrics
    ↓
    return CheckMeshResult
```

## 📈 성능 특성

### NativeMeshChecker 성능
- **처리 시간**: polyMesh 로드 + 계산 < 1초
- **메모리**: numpy 배열 (O(n_cells + n_faces))
- **스케일**: 250k face 메시도 처리 가능

### Neatmesh 성능
- **추가 시간**: ~0.5-2초 (메시 크기에 따라)
- **메모리**: 메시 + analyzer 객체
- **실패 대응**: Try/catch로 무시 가능

## 🔄 사용 방법

### OpenFOAM이 설치된 경우
```python
from core.evaluator.quality_checker import MeshQualityChecker

checker = MeshQualityChecker()
result = checker.run(case_dir)  # checkMesh 사용
```

### OpenFOAM이 없는 경우 (자동 fallback)
```python
from core.evaluator.quality_checker import MeshQualityChecker

checker = MeshQualityChecker()
result = checker.run(case_dir)  # NativeMeshChecker + neatmesh 사용
```

**사용자 관점**: 투명한 fallback (코드 변경 없음)
